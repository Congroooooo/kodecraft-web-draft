# KodeCraft Website Creative Context

## Project Overview

We are redesigning the official KodeCraft company website into a premium recruiting-focused web presence. The goal is to make the site feel like the front door of a serious engineering company: polished, technical, confident, and human enough to attract top-tier builders.

The website should communicate that KodeCraft is not a generic outsourcing or body-shop company. It is a team of product-minded engineers building real systems, agent-native products, Rust infrastructure, MCP tooling, automation, full-stack products, and high-quality internal or client-facing software.

The core message:

> Where serious builders shape what comes next.

## Brand Personality

KodeCraft should feel:

- Technical but not cold
- Premium but not corporate
- Futuristic but practical
- Sharp, disciplined, and engineering-led
- Human, collaborative, and craft-oriented
- Confident without being arrogant
- Built for serious developers, designers, and product-minded contributors

This website is meant to attract people who care about ownership, engineering judgment, clean architecture, performance, product craft, and meaningful work.

## Visual Theme

The current visual direction is a dark, cinematic, AI-infrastructure-inspired website with a green brand accent.

The visual language should feel like:

- Deep dark interface backgrounds
- Subtle green energy/glow
- Technical grid systems
- Data-flow motion
- Neural networks and agent systems
- Clean code and systems architecture
- Premium SaaS/infrastructure design
- Minimal, precise, high-contrast composition

The design should avoid feeling like a generic AI startup template. It should feel intentional, restrained, and custom to KodeCraft.

## Color Palette

Use the existing KodeCraft branding colors and the current website direction.

Primary colors:

- Near-black / deep navy background: `#070B14`
- Dark surface: `#0A0F1A`
- Slate-blue panels: `#141E30`
- Primary brand green: `#4ADE80`
- Secondary green accent: `#8BC53F`
- Primary text: `#F0F5FF`
- Muted text: `#94A3B8`

General color guidance:

- Backgrounds should stay dark and premium.
- Green should be used as a focused brand signal, not as a full-page wash.
- Avoid purple/blue AI gradients.
- Avoid beige, brown, orange, or playful palettes.
- Keep contrast strong and readable.
- Green lighting should feel like subtle technical energy, not neon decoration.

## Typography

The site uses a Geist-inspired typography direction.

Typography should feel:

- Clean
- Modern
- Technical
- Highly readable
- Tight and premium in headlines
- Calm and structured in body copy

Headline style:

- Large but controlled
- Strong weight
- Tight line-height
- Slight negative tracking
- Clear line breaks

Body style:

- Muted slate text
- Comfortable line-height
- Not too small
- Easy to scan

Avoid decorative, handwritten, playful, or overly futuristic sci-fi fonts.

## Hero Section Direction

The hero is the signature moment of the site.

It uses a cinematic background video showing data flow, neural-network visuals, and a walking digital silhouette. The hero includes:

- KodeCraft logo
- Navigation: Work, Culture, Roles, Contact
- CTA: Apply Now
- Main headline: “Where serious builders / shape what comes next.”
- Supporting copy about agent-native products and Rust infrastructure
- CTA buttons: “See Open Roles” and “Explore the Work”
- Right-side proof text: “Live Systems / Rust. Agents. Ownership.”

The hero should feel immersive, premium, and technical. Any generated asset used near the hero should complement the existing video, not compete with it.

Hero asset guidance:

- Dark background
- Data particles
- Neural systems
- Abstract agent infrastructure
- Subtle green illumination
- Strong negative space
- No obvious stock-photo look
- No cartoon characters
- No generic robot imagery
- No busy dashboard screenshots unless intentionally stylized

## “Not a Body Shop” Section

This section is a bold text-driven storytelling moment.

Message:

> This is not a body shop. We build products, platforms, and agent-native systems where engineers shape what gets built, how it is architected, and why it matters.

The section uses a dark static technical grid background and a premium scroll-based text highlight animation:

- Text starts dark gray
- As the user scrolls, words progressively turn bright white
- The reveal follows natural reading order
- Scrolling back reverses the highlight
- Only text color animates
- The background remains static

This section should feel editorial, serious, and declarative.

## Motion Direction

Motion should feel premium and intentional.

Approved motion patterns:

- Scroll-scrubbed text highlighting
- Subtle reveal-on-scroll
- Soft hover lift on buttons/cards
- Slow scanline movement in technical sections
- Video-synced ambient hero glow
- Smooth color and opacity transitions

Avoid:

- Bouncy motion
- Excessive blur
- Random floating shapes
- Overly fast particle effects
- Distracting looping animations
- Motion that makes text harder to read

Motion should guide attention, not decorate for its own sake.

## Asset Generation Goals

The generated asset should support the website’s premium engineering identity.

Good asset directions:

- Cinematic dark technical environment
- Abstract neural/data architecture
- Futuristic engineering blueprint
- Agent system visualization
- Human silhouette made of data or code
- Clean product/system diagram
- Subtle green glow on deep navy background
- High-contrast but restrained composition
- Strong empty space for text overlays

Bad asset directions:

- Generic AI robot
- Cartoon mascot
- Overly colorful tech illustration
- Purple/blue gradient blob background
- Stock corporate people shaking hands
- Busy server-room cliche
- Cyberpunk neon overload
- Random code rain
- Decorative orbs or bokeh blobs

## Suggested Prompt For Asset Generation

Create a premium dark technical visual for the KodeCraft company website. The brand is a serious engineering company building agent-native products, Rust infrastructure, MCP tooling, automation, and full-stack systems. Use a deep navy-black background with subtle KodeCraft green illumination. The image should feel cinematic, minimal, intelligent, and systems-oriented. Include abstract data flow, neural network structure, or clean architecture-like connections. Keep strong negative space for website text. Avoid robots, stock-photo styling, colorful gradients, cartoon elements, and noisy cyberpunk effects. The mood should be premium, futuristic, precise, and engineering-led.

## Practical Asset Requirements

Recommended formats:

- Hero or section background: `1920x1080` or wider
- Tall section visual: `1600x2000`
- Card/feature visual: `1200x900`
- Transparent overlay asset: PNG or WebP with alpha if possible
- Background video or motion asset: WebM preferred

Implementation preferences:

- Assets should work on dark backgrounds.
- Important visual details should not sit at the far edges.
- Leave safe empty space for text on the left when used in hero-like sections.
- Avoid tiny text inside generated images unless it is abstract and not meant to be read.
- Keep file sizes web-friendly.

The KodeCraft website should make a strong engineer think:
